Primeiramente, criamos uma pasta para o projeto e abrimos o Git Bash dentro dela. 

Em seguida, inicializamos um repositório local utilizando o comando git init, 
permitindo que o Git começasse a monitorar as alterações realizadas nos arquivos.

Depois disso, acessamos o GitHub e criamos um repositório remoto para armazenar o 
projeto online. Com o repositório criado, conectamos o projeto local ao remoto 
utilizando o comando git remote add origin, seguido do link fornecido pelo GitHub. 
Também realizamos a clonagem do projeto utilizando o comando git clone, o que 
permitiu criar uma cópia completa do repositório em outro local. Essa etapa foi 
importante para entendermos como diferentes integrantes podem acessar e trabalhar
no mesmo projeto.

Durante o desenvolvimento, criamos páginas HTML simples e utilizamos branches
para dividir as tarefas. Criamos uma nova branch com o comando git branch e
acessamos ela utilizando git checkout. Dessa forma, cada integrante pôde 
realizar alterações separadamente, sem modificar diretamente a versão principal 
do projeto. 

Conforme as alterações eram feitas, utilizávamos o comando git add .
para adicionar os arquivos modificados e, em seguida, registrávamos as mudanças
com git commit -m, descrevendo o que havia sido alterado. Depois disso, 
enviávamos as atualizações para o GitHub utilizando o comando git push. 

Após concluir as modificações necessárias, realizamos o merge das branches para 
juntar todas as alterações em uma única versão do projeto. Para isso, retornamos à 
branch principal utilizando git checkout main e executamos o comando git merge.
Esse processo permitiu integrar o trabalho realizado por cada integrante da equipe.

Por fim, fizemos uma integração simples entre duas páginas HTML, adicionando links para que
fosse possível navegar de uma página para outra. Após realizar alguns testes e verificar 
que tudo estava funcionando corretamente, enviamos a versão final para o GitHub.
Com essa atividade, foi possível compreender melhor a importância do Git e do GitHub no
desenvolvimento de sistemas. Além de facilitar a organização do projeto, essas ferramentas
tornam o trabalho em equipe mais eficiente, permitindo o acompanhamento das alterações e a
integração das contribuições de cada integrante.
